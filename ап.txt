git status — отображает текущее состояние репозитория - показывает состояния файлов в рабочей директории и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе.

git add <нужный файл> — добавить измененный/новый файл (индексирует), в последующие изменения (коммит)

git commit -m "<ваше сообщение с описанием>" — фиксирует проиндексированные изменения и сохраняет их, а также добавляет описание изменения

git push — отправить все изменения на удаленный репозиторий

cd .. — вернуться на директорию выше

mkdir <название папки> — создать новую папку в папке, в которой находимся

cd <название папки> — переход в нужную папку

git clone https://github.com/<ваш github>/<репозиторий>.git — клонирование удаленного репозитория себе локально

ls — вывести названия файлов и директорий

git pull — загрузить изменения на локальный репозиторий (по всем веткам)
